# Day 06 - SIEM Tools and Network Monitoring

## Event Logs

### Log

A record of events that occur within an organization's systems.

Logs help security teams:

* Monitor activity
* Detect suspicious behavior
* Investigate security incidents
* Track system changes

Logs are one of the most important sources of information for security analysts.

---

## Security Information and Event Management (SIEM)

### SIEM Tool

A Security Information and Event Management (SIEM) tool is an application that collects and analyzes log data to monitor critical activities within an organization.

SIEM tools help security teams:

* Collect security data
* Analyze events
* Detect threats
* Investigate incidents
* Monitor organizational systems

---

## Common SIEM Tools

### Splunk

A data analysis platform used to collect, search, and analyze large amounts of machine-generated data.

Security teams use Splunk to:

* Analyze logs
* Investigate incidents
* Monitor security events

### Google Chronicle

A cloud-native SIEM tool that stores security data for search and analysis.

Google Chronicle helps organizations:

* Store large amounts of security data
* Search security events quickly
* Investigate threats efficiently

---

## Other Security Tools

### Playbook

A manual that provides details about operational actions and incident response procedures.

Playbooks help analysts:

* Follow consistent procedures
* Respond to incidents effectively
* Investigate alerts step-by-step

### Network Protocol Analyzer (Packet Sniffer)

A tool designed to capture and analyze data traffic within a network.

Network protocol analyzers help security professionals:

* Monitor network traffic
* Troubleshoot network issues
* Detect suspicious activity
* Investigate security incidents

## Personal Takeaways

* Logs are critical because they provide evidence of what is happening within a system.
* SIEM tools help security analysts collect and analyze large amounts of security data.
* Splunk and Google Chronicle are two commonly used SIEM platforms.
* Playbooks provide a structured process for responding to incidents.
* Packet sniffers allow analysts to see and analyze network traffic, making them valuable tools for investigations.
