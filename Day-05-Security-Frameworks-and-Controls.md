# Day 05 - Introduction to Security Frameworks and Controls

## Role of a Security Analyst

One of the main responsibilities of a Security Analyst is to identify and analyze an organization's:

* Assets
* Risks
* Threats
* Vulnerabilities

By understanding what needs protection and the risks involved, analysts can help organizations build stronger security strategies.

---

## Security Frameworks

### Security Framework

Security frameworks are guidelines used for building plans to help organizations mitigate risks and protect data and privacy.

Security frameworks help organizations:

* Identify security risks
* Create security policies
* Improve security practices
* Protect critical systems and information

---

## Four Core Components of Frameworks

Effective security frameworks contain four main components:

### Communication

* Ensures security information is clearly shared across the organization.

### Process

* Provides step-by-step procedures for handling security activities.

### Guidelines

* Establishes recommended practices and standards.

### Goals

* Defines what the organization wants to achieve with its security strategy.

---

## CIA Triad

The CIA Triad is a foundational cybersecurity model used to help organizations manage risks and create security policies.

CIA stands for:

### Confidentiality

* Ensures that information is only accessible to authorized individuals.

### Integrity

* Ensures that data remains accurate, trustworthy, and unchanged unless properly authorized.

### Availability

* Ensures that systems, applications, and data are accessible when needed by authorized users.

---

## NIST Cybersecurity Framework (CSF)

The NIST Cybersecurity Framework is a voluntary framework that provides standards, guidelines, and best practices for managing cybersecurity risk.

Organizations use the NIST CSF to:

* Identify cybersecurity risks
* Protect systems and data
* Improve their overall security posture
